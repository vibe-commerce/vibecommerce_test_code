# Auto-Backup Rule

После завершения значимой работы — **предложи пользователю сделать бэкап** в GitHub.

## Что считается значимой работой

Предлагай `/backup` после:

1. **Завершение задачи** — создание/редактирование файлов, обновление бэклога, изменение структуры
2. **Объёмный план** — декомпозиция проекта, планирование спринта, формирование стратегии
3. **Работа с Excel** — обновление бэклога, создание/изменение отчётов и моделей
4. **Создание документации** — новые README, обновление архитектурных документов, описания проектов
5. **Изменение конфигурации** — новые скиллы, правила, промпты, роли
6. **Множественные изменения** — если за сессию изменено 3+ файла

## Что НЕ является значимой работой

НЕ предлагай бэкап после:
- Только чтения и анализа (без изменений файлов)
- Ответов на вопросы без редактирования
- Незавершённой работы (пользователь явно продолжает)

## Формат предложения

После завершения значимой работы добавь в конец ответа:

```
---
Сделать бэкап в GitHub? Скажи "да" или `/backup` — я закоммичу и запушу.
```

## Важно

- НЕ делай бэкап автоматически — только предлагай
- Пользователь сам решает, когда пушить
- Если пользователь подтвердил — выполни скилл `/backup`
