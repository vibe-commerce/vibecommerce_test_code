## Обзор сервиса

**MP Manager** (mpmgr.ru) — платформа для управления рекламой, ценами и аналитикой на маркетплейсах.

### Поддерживаемые маркетплейсы

| Маркетплейс | Автоцены | Автоставки | Аналитика | Автоответы |
|-------------|----------|------------|-----------|------------|
| Wildberries | ✅ | ✅ | ✅ | ✅ |
| Ozon | ✅ | ✅ | ✅ | ✅ |
| Яндекс.Маркет | ✅ | ✅ | — | ✅ |

### Ключевые возможности

1. **Автоцены (Репрайсер)** — автоматическая корректировка цен по правилам, конкурентам, остаткам, маржинальности
2. **Менеджер цен** — индивидуальные цены по размерам, кросс-цена между площадками
3. **Автоставки** — управление рекламными ставками с AI-оптимизацией
4. **Автоответы на отзывы** — AI-генерация персональных ответов клиентам
5. **Внешняя аналитика** — 100+ отчётов с 70-150 параметрами
6. **SEO** — мониторинг позиций, автооптимизация карточек
7. **Поставки** — планирование доставок, поиск слотов, анализ тарифов

### Доступ к сервису

- Веб-версия: [app.mpmgr.ru](https://app.mpmgr.ru)
- PWA-приложение (устанавливается из браузера на iOS/Android)
- Расширение для браузера

## Подключение маркетплейсов

### Шаг 0: Регистрация и создание организации

1. Регистрация на [app.mpmgr.ru](https://app.mpmgr.ru) через Email или Яндекс-аккаунт
2. Создание организации:
   - Название компании
   - Имя сотрудника
   - Тип: российское / зарубежное представительство
   - Опционально: реферальный код (даёт 5 дней пробного периода вместо 3)

> **Важно:** Входить нужно тем же способом, что использовался при регистрации. Если зарегистрировались через email @yandex.ru, нужно вводить email, а не нажимать "Войти с помощью Yandex".

---

### Wildberries

**3 способа подключения:**

#### Способ 1: Одноклик из сервиса (рекомендуемый)
1. В MP Manager → Настройки → «Подключить Wildberries»
2. Выбрать организацию
3. Разрешить доступы в кабинете WB

#### Способ 2: Через API ключ
1. Кабинет поставщика WB → Профиль → Настройки → Доступ к новому API
2. «Создать новый токен» → тип: «Для сервиса из каталога» → найти MP Manager
3. **Обязательно:** выбрать права «Чтение и запись» (не только чтение!)
4. Скопировать ключ → вставить в MP Manager

#### Способ 3: Через «Авторизованные сервисы»
1. Страница авторизованных сервисов WB → найти MP Manager
2. Подтвердить разрешения

**Необходимые права доступа по функциям:**

| Функция | Требуемые права |
|---------|----------------|
| Автоставки | Продвижение, Контент, Статистика, Аналитика |
| Автоцены | Контент, Цены |
| Аналитика | Статистика, Аналитика, Контент |
| Автоответы | Вопросы/отзывы, Контент |
| Мониторинг | Не требуется (только Контент для отображения) |

**Замена ключа:** удалить текущий → сохранить → ввести новый → сохранить.

---

### Ozon

#### Шаг 1: API ключ (Seller API)
1. Ozon Seller → Профиль → Настройки → Seller API
2. «Сгенерировать ключ»
3. Название: `mpmgr`
4. **Тип токена: Admin** (обязательно)
5. Скопировать ключ → вставить в MP Manager

#### Шаг 2: ID поставщика
Найти на странице настроек Ozon, вкладка «API ключи».

#### Шаг 3: Client ID и Client Secret (для рекламы)
1. Профиль → Настройки → API ключи → вкладка «Performance API»
2. «Создать аккаунт» → добавить ключ
3. Скопировать Client ID и Client Secret → вставить в MP Manager

#### Шаг 4: Добавление бота MP Manager (для автоответов/МРЦ)
1. Ozon Seller → Сотрудники → «Пригласить сотрудника»
2. Ввести email из настроек MP Manager
3. Роль: **«Менеджер по товарам»**
4. Дождаться подтверждения (обычно в течение суток)

**Необходимые роли по функциям:**

| Функция | Роли |
|---------|------|
| Ответы на отзывы | Бот + Менеджер по товарам |
| Удержание МРЦ | Менеджер по товарам + Финансовый менеджер или Технологический партнёр |
| Автоставки | Бот + Менеджер по товарам |
| Просмотр ключевых слов | Бот + Аналитик |

---

### Яндекс.Маркет

#### Шаг 1: API ключ (авторизационный токен)
1. Кабинет ЯМ → Модули и API → Авторизационный токен → «Создать»
2. Указать название токена
3. Выбрать доступы (рекомендуется: **«Полное управление кабинетом»**)

**Минимальные доступы по функциям:**

| Функция | Доступы |
|---------|---------|
| Автоответы | Общение с покупателями, Просмотр товаров и карточек |
| Автоставки | Продвижение товаров, Просмотр информации о продвижении, Просмотр товаров |
| Автоцены | Просмотр товаров, цен, управление ценами, информация о заказах |

#### Шаг 2: ID поставщика (Business ID)
Два способа найти:
- Модули и API → блок «Передача данных Маркету» → ID кабинета (business_id)
- Товары → Каталог → переключатель аккаунтов в правом верхнем углу

#### Шаг 3: Вставить данные
Вставить API ключ и Business ID в настройки организации в MP Manager.

## Стратегии автоцен (Wildberries)

MP Manager предлагает **5 стратегий** автоматического управления ценами на Wildberries.

### 1. По правилам

**Суть:** Гибкое ценообразование по набору условий (if/then). Самая мощная и настраиваемая стратегия.

**Как работает:**
- Задаёте базовую цену (текущая или своя)
- Создаёте правила с фильтрами (условия) и действиями (изменение цены)
- Система проверяет фильтры и выполняет действия

**Компоненты правила:**

| Элемент | Описание |
|---------|----------|
| Базовая цена | Опорная точка: текущая цена или пользовательская |
| Фильтры | Условия срабатывания: «если» / «если не» |
| Действия | Понизить (₽/%), Повысить (₽/%), Задать значение |
| Мин/макс цена | Границы — система не выйдет за них |
| Режим продолжения | «Завершить» (стоп) или «Продолжить» (следующее правило) |

**Доступные фильтры:**
- Остатки на складах
- Количество заказов
- Оборачиваемость за период
- CTR рекламных кампаний
- Конверсия в корзину / в заказ
- Позиция в поисковой выдаче
- Другие метрики карточки

**Пример:** Если остатков < 50 шт → повысить цену на 5%. Если остатков > 500 шт → понизить на 10%.

**Документация:** [docs.mpmgr.ru/docs/wildberries/auto-prices/rules/](https://docs.mpmgr.ru/docs/wildberries/auto-prices/rules/)

---

### 2. По конкурентам

**Суть:** Автоматическая корректировка цен относительно конкурентов.

**Как работает:**
1. Добавляете конкурентов (ссылки или артикулы, можно списком)
2. Настраиваете автофильтры (остатки, рейтинг, отзывы)
3. Выбираете режим отслеживания и действие

**Режимы отслеживания:**

| Режим | Описание |
|-------|----------|
| По списку (сверху вниз) | Проверяет конкурентов по порядку |
| Минимальная цена | Берёт самую низкую среди конкурентов |
| Максимальная цена | Берёт самую высокую среди конкурентов |
| Средняя цена | Рассчитывает среднюю по всем |

**Действия:** Понизить на ₽/%, Повысить на ₽/%, Задать фиксированную.

**Автофильтры конкурентов:**
- Остатки на складах
- Рейтинг товаров
- Количество отзывов
- Логика: фильтры внутри группы — «И», между группами — «ИЛИ»

**Документация:** [docs.mpmgr.ru/docs/wildberries/auto-prices/competitor/](https://docs.mpmgr.ru/docs/wildberries/auto-prices/competitor/)

---

### 3. Удержание МРЦ с учётом СПП

**Суть:** Удерживает финальную цену для покупателя на целевом уровне, компенсируя скидки WB (СПП и кошелёк).

**Три режима:**

| Режим | Что учитывает |
|-------|---------------|
| Без учёта СПП | Стандартная цена со скидкой продавца |
| С учётом СПП | Корректирует цену так, чтобы после вычета СПП она была целевой |
| С учётом СПП + кошелёк WB | Учитывает обе скидки при расчёте |

**Параметры:**
- Целевая цена (МРЦ) — точная сумма для удержания
- Мин/макс цена — границы диапазона
- Интервал обновления
- Расписание по дням/часам

**Ключевое ограничение:** На маркетплейсе одна цена на товар. Сервис выбирает ПВЗ и платформу для расчёта (по умолчанию: Android, Москва).

**Документация:** [docs.mpmgr.ru/docs/wildberries/auto-prices/mrc/](https://docs.mpmgr.ru/docs/wildberries/auto-prices/mrc/)

---

### 4. Кросс-цена (WB ↔ Ozon)

**Суть:** Синхронизация цен между Wildberries и Ozon — удержание одинаковой цены на обеих площадках.

**Как работает:**
1. Выбираете, какая цена управляемая (WB или Ozon)
2. Привязываете товар на второй площадке
3. Система автоматически выравнивает цены

**Параметры:**
- Выбор управляемой цены
- Товар-привязка на Ozon
- Режимы СПП (как в стратегии МРЦ)
- Мин/макс цена, расписание

**Документация:** [docs.mpmgr.ru/docs/wildberries/auto-prices/cross-price/](https://docs.mpmgr.ru/docs/wildberries/auto-prices/cross-price/)

---

### 5. Удержание маржинальности

**Суть:** Автоматически рассчитывает и удерживает цену, обеспечивающую целевой уровень маржи.

**Параметры:**

| Параметр | Описание |
|----------|----------|
| Интервал сбора статистики | Период для расчёта маржинальности |
| Процент маржинальности | Целевой уровень прибыли |
| Тип комиссии | Какие сборы учитывать |

**Режимы:**
- Без СПП
- С учётом СПП
- С учётом СПП + кошелёк WB

**Защита:** Барьер 20% — резкие изменения требуют подтверждения. Строго в рамках мин/макс цены.

**Документация:** [docs.mpmgr.ru/docs/wildberries/auto-prices/marginality/](https://docs.mpmgr.ru/docs/wildberries/auto-prices/marginality/)

## Стратегии автоцен (Ozon)

### 1. По правилам

Аналогично Wildberries, но с особенностями:
- **Управление только ценой** (скидка продавца недоступна)
- Те же компоненты: базовая цена, фильтры, действия, мин/макс
- Расписание по дням и часам
- Защита от скачков >20%

**Документация:** [docs.mpmgr.ru/docs/ozon/auto-prices/rules/](https://docs.mpmgr.ru/docs/ozon/auto-prices/rules/)

### 2. Удержание МРЦ с учётом карты Ozon

**Суть:** Удерживает финальную цену для покупателя с учётом скидок по карте Ozon.

**Особенности:**
- **Обязательно** подключить бота MP Manager (см. подключение Ozon, шаг 4)
- Скидки Ozon могут быть 0% → минимальная цена должна совпадать с целевой МРЦ
- Защита от скачков >20%

**Документация:** [docs.mpmgr.ru/docs/ozon/auto-prices/mrc/](https://docs.mpmgr.ru/docs/ozon/auto-prices/mrc/)

## Общие настройки репрайсера

Все стратегии имеют набор общих параметров.

### Базовые параметры

| Параметр | Описание |
|----------|----------|
| Название | Произвольное имя стратегии |
| Интервал обновления | Как часто проверяются и корректируются цены |
| Управление | Цена или скидка продавца (Ozon — только цена) |
| Мин. цена | Нижняя граница — система не опустит ниже |
| Макс. цена | Верхняя граница — система не поднимет выше |

### Расписание

- Разные стратегии для каждого дня недели
- Гранулярность по часам (0-23)
- Пример: будни — одна цена, выходные — другая

### Защита от скачков

- При изменении цены >20% за раз — система не применяет автоматически
- Предотвращает ошибки из-за некорректных данных маркетплейса

### Выбор ПВЗ и платформы (WB)

- На WB одна цена на товар, но СПП различается по регионам и платформам
- Сервис позволяет выбрать ПВЗ и платформу (Android/iOS/Web)
- По умолчанию: **Android, Москва**

### Диагностика: почему цена не меняется?

1. Сработала защита от скачков (>20%)
2. Цена вышла за пределы мин/макс
3. Некорректная базовая цена
4. Нет подходящих правил для текущих условий
5. Интервал обновления ещё не прошёл
