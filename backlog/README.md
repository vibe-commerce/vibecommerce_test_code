# Бэклог задач

Last Updated: 2026-02-13

## Описание
Папка для хранения задач по всем проектам.

## Статус
Бэклог в процессе формирования. Файл `Backlog.txt` будет создан при первом добавлении задач через `/project-manager`.

## Формат Backlog.txt
Простой текстовый файл. Каждая задача содержит поля:
- n — уникальный ID
- PRJ — код проекта
- Название задачи — action-oriented заголовок
- Описание задачи — что сделать
- Приоритет — 1–100
- Дедлайн — YYYY-MM-DD
- Спринт — дата окончания спринта
- Статус — TO DO / WIP / ON PRIORITY / DONE / ON HOLD
- Комментарий — последний апдейт

## Workflow
1. Новые задачи добавляются в `Backlog.txt`
2. `/project-manager` управляет приоритизацией
3. Работа по спринтам
